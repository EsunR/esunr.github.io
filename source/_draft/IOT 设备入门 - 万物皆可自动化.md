# 1. 前言

当你在温暖的被窝不想起身关灯时，当你周末想睡个懒觉却被震耳欲聋的门禁吵醒时，当你在夜里十一点下班到家开门摸黑换鞋时，当你在寒冷的半夜挣扎着起床上厕所却发现被室友占用时...

你有没有想过难道这些我们每天困扰着我们的事情难道不能自动化吗？我们可以在被窝里直接用手机控制灯的开关，周末门禁响了就自动接听，当我们下班快到家的时候空调和玄关的灯会自动开启，我们半夜起来上厕所前看一眼手机就知道厕所有没有人。

这些并非不可实现的，我们只需要一点点的动手能力和钞能力，打开购物软件，搜索米家就能找到一堆智能家庭设备。

作为普通人，只需要给雷总充钱就能享受到很多 IOT 设备的乐趣了，发挥自己的想象力就可以搞定各种各样场景的自动化。但是如果这次分享只告诉你要买什么设备就太无趣了，我要向大家分享的如何在现有的框架上去开发这些设备获取更多的能力，并且 IOT 设备的入门门槛并非很高，你完全可以用自己的代码驱动这些设备，甚至是手搓一个适合自身场景的智能设备，这些东西其实就像搭积木一样简单。

# 2. 准备工作 - 升级你的家庭终端

再买了一堆家庭智能设备之后，你必定会遇到跨品牌的设备，比如你家里的大多数设备都是米家的，然而热水器是海尔的、落地灯是小度的，这样不仅用着麻烦，并且不同的设备因为品牌之间的隔离也没法互动。

但这并非无解，IOT 设备能在局域网进行控制必定是在发送某种通信报文，我们不一定非得用 APP 来控制设备，只要知道设备是如何通信的就可以了。那有没有一个平台可以聚合所有品牌智智能设备的通信，我们只需要通过这个平台来调用家里的智能设备就能实现统一？那必定有的，这个平台就是 Homeassistant（后面简称 HA）。

![image.png|400](https://s2.loli.net/2025/12/22/1ER2eJO7AVXrltL.png)

HA 是一个平台，其强大的插件机制给平台提供了横向扩展的能力，开发者只需要编写一个可以与设备交互的插件，就能轻松接入到 HA，并且 HA 还提供了自动化、脚本、HTTP API 等能力来驱动接入的设备。其影响力之大，甚至有些智能家居的官方甚至都会为其编写插件，比如小米官方的 [ha_xiaomi_home](https://github.com/XiaoMi/ha_xiaomi_home)、易微联的 [eWeLink HA Add-on](https://sonoff.tech/blogs/news/how-sonoff-works-with-home-assistant?srsltid=AfmBOoriu4tV5Tk5PmjvVGmpUrXp48YG12405R9T3ka0YXSp1aT9YIKB)，并且例如 Metter 这样新的智能家居连接标准其都会在第一时间进行实现。

HA 在局域网中需要一个运行载体，这个载体只要可以运行 Docker 即可。如果你有软路由设备、或者你的路由器支持 Docker、亦或是任何可以 24 小时运行的 Linux 设备，将其部署到 Docker 上即可。如果是从零开始，可以买一个树莓派或者香橙派这些很便宜的单板计算机来运行 HA，但是个人更建议使用 X86 去运行 HA，你可以在闲鱼上搜索 DIY NAS，或者愿意花更多钱的话就买一台 N100、N150 这种低功耗的小主机（然后你后面就可以折腾另外一个更大的坑 - NAS）。

> 如果真的想一分钱不出的话，你废旧的手机其实也可以跑 Linux 并安装 Arm 版的 HA：https://blog.esunr.site/2021/12/abf074494c8f.htm

# 3. 小试牛刀 - 初试 HA

### 米家设备接入 HA

由于开源社区上的插件并不会进入 HA 官方插件市场，所以玩 HA 的话必须去安装一个 HASC 第三方插件市场才能安装在 Github 上发布的插件，米家的插件就需要使用 HASC 进行安装。

首先按照官方文档安装 HACS：[https://www.hacs.xyz/docs/use/](https://www.hacs.xyz/docs/use/)

然后按照米家集成的说明对米家集成进行安装：[https://github.com/XiaoMi/ha_xiaomi_home](https://github.com/XiaoMi/ha_xiaomi_home)

安装完成之后点击添加中枢，输入小米的账号后就会自动同步小米账号中的所有设备到 HA：

![image.png|600](https://s2.loli.net/2025/12/22/i2kWfXObLTYvSBd.png)

HA 中的**设备**是由多个开关（Switch）、传感器（Sensor）、按钮（Button）等**实体**组成的，通过控制这些实体就可以控制设备进行各种各样的功能：

![image.png|600](https://s2.loli.net/2025/12/22/7Cb4RJBAWG3z6kH.png)


### 接入 HomeKit

不知道大家有没有留意到过 iOS 中的家庭 APP，我认为其是最好用的智能家居控制 APP，干净、没有广告、UI 好看、功能强大。如果你是苹果生态，并且家里有吃灰的 Homepod 或者 AppleTV，借助 HA 你可以将刚才上一步添加的米家设备接入到苹果家庭 APP 中了。

> Homepod 和 AppleTV 起到的是 HomeKit 家庭中枢的作用，让用户脱离家庭局域网后仍然能控制设备。

接入 HomeKit 可以大大的减少我们操作家庭智能设备的步骤，比如直接在控制中心操控家庭设备，而不需要打开各个品牌的 APP。同时可以直接对接苹果的自动化，比如基于地理位置的电子围栏。更重要的是可以帮我们剩下一大笔置换 HomeKit 设备的费用。

[HomeKit Bridge](https://www.home-assistant.io/integrations/homekit) 是 HA 的一个官方集成，只要你家中有 HomeKit 中枢 HA 就会自动推荐该插件给你，直接安装即可：

![image.png|400](https://s2.loli.net/2025/12/22/jBCnv1dy4NZKmEs.png)

在配置过程中简单的挑选你想加入到 Homekit 的设备，这里推荐使用 bridge 模式，你可以以设备类型为纬度或者设备区域为纬度创建桥，其会生成一个桥接设备，我们只需要向桥接设备添加 HA 设备即可，方便我们管理：

![image.png|400](https://s2.loli.net/2025/12/22/KXy4b3HaN2ZIDAV.png)

然后通知中心就会出现一个二维码，用家庭 APP 扫描二维码即可完成添加：

![image.png|400](https://s2.loli.net/2025/12/22/mCkUe2r1cKFhxsd.png)
